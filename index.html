PocketJS 是什么

一个跑在游戏掌机上的 UI 框架

写法是前端的

JSX + Tailwind 子集，样式、字体、动画全部在编译期烘焙成二进制。

运行时是系统级的

Rust core 负责布局与渲染，直接生成 GPU 显示列表。

框架层可插拔

Solid 与 Vue Vapor 是两个一等公民，驱动同一棵原生树。

目标机器只有 333 MHz 和 32 MB 内存，但要跑满 60 fps。

掌机上的 Vue 应用

import { onMounted, ref } from "vue";
import { mount } from "@pocketjs/framework/vue-vapor";
import { View, Text } from "@pocketjs/framework/vue-vapor/components";

export default function App() {
  const count = ref(0);
  onMounted(() => console.log("mounted on a real PSP"));

  return (
    <View class="p-4 flex-col gap-2">
      <View focusable onPress={() => count.value++}>
        <Text class="text-sm text-blue-600">Count: {count.value}</Text>
      </View>
      {count.value > 2 ? <Text>Vue Vapor, native tree.</Text> : null}
    </View>
  );
}
mount(App);

不是"像 Vue"—— import 的就是 vue：真的 ref，真的 onMounted，真的 Vapor 编译。

为什么是 Vapor

vdom 时代：自定义渲染器

组件
↓
虚拟 DOM 树
↓
diff / patch 协调运行时
↓
宿主节点接口

Vapor：编译产物直达节点

组件
↓ 编译期展开
createElement / insert / setProp
↓
PocketJS 原生树

把 Vue 接上 PSP 的渲染绑定：244 行  （渲染器 108 + DOM 门面 136；整个 Vue 适配层 < 1200 行）

一次 count.value++ 的旅程

JAVASCRIPT
ref 变更 → Vapor effect 触发
setProp(节点镜像, "text", …)
RUST CORE
指令流 ops → 布局 / 二进制样式表
GPU
sceGu 显示列表 → 屏幕
没有 虚拟 DOM
没有 diff
没有 DOM 本身

一次状态变更 = 一条精确的写指令。
这就是它塞得进 32 MB 的原因。

Vapor 不只是让 Vue 变快，
它让"渲染到哪里"
第一次变得便宜。

薄框架，厚运行时

浏览器替你做的事，得有人重做一遍

编译期烘焙

Tailwind 子集 → 二进制样式表；字体 → 图集；关键帧 → 烘焙时间轴，运行时零计算。

运行时系统

输入与焦点、覆盖层、动画引擎、.pak 资产容器、PSP / Vita 原生构建。

这些全部与框架无关 —— 换框架只换响应式层和那 244 行绑定，系统层原封不动。

动效

动画也在编译期

关键帧在构建期烘焙成二进制时间轴，播放时零 JS 参与 —— 相当于把 CSS 动画的思路推到掌机上。

Pocket Figma

整份 Figma 设计稿，搬上掌机

瓦片化流式渲染 + 调色板纹理。DOM 消失之后，才看清 DOM 替你付过多少钱。

同一份代码，两台真机

两倍像素，零分叉

同一份应用跑 PSP 和 PS Vita；OpenStrike 用同一套运行时思路在真机锁 60 fps 跑 3D —— 帧预算是硬约束时，细粒度是唯一活法。

Pocket YouTube

在 PSP 上看 YouTube

Mac 出算力，掌机出画面，USB 当总线。运行时保持小而硬核，能力靠主机协作补齐。
